## 1. 技术架构与实现方案

### 1.1 技术栈选型矩阵

| 层级 | 技术组件 | 选型理由 | 替代方案 |
|------|----------|----------|----------|
| **前端框架** | **Next.js 14 (React)** | SSR利于SEO，路由灵活，Vercel一键部署 | Nuxt.js, Remix |
| **3D/动效** | **React Three Fiber + Drei** + **Framer Motion** | React生态内最成熟的WebGL方案，支持PBR材质 | Three.js原生, GSAP |
| **状态管理** | **Zustand** | 轻量级，适合摇卦状态、音频、AI流式输出管理 | Redux, Jotai |
| **后端/DB** | **Supabase (PostgreSQL)** | Serverless，自带Auth，免费额度足，支持Edge Functions | Firebase, MongoDB |
| **AI 接口** | **Claude 3.5 Sonnet API** | 文学性与灵性语境理解力最强，适合解卦 | OpenAI GPT-4o, Gemini |
| **支付系统** | **Stripe Checkout** | 支持全球信用卡/Apple Pay，合规性好，无需自建支付流 | Lemon Squeezy |
| **部署平台** | **Vercel** | Next.js官方托管，全球CDN，自动HTTPS | Netlify, AWS |

### 1.2 核心架构设计

```
[用户浏览器]
    ↓ (HTTPS)
[Cloudflare CDN / Vercel Edge]
    ↓
[Next.js SSR 服务端] ──── [Supabase (用户数据/订单)]
    ↓                           ↑
[Three.js 3D渲染] ──── [Stripe (支付回调)]
    ↓
[Claude API (AI解文)] ──── [Prompt模板库]
```

**技术亮点：**
- **首屏加载优化**：3D模型使用 `.glb` 格式（<5MB），启用 `Draco` 压缩。首屏采用静态星空背景图（WebP），3D场景异步加载，确保 LCP < 1.5s。
- **AI流式输出**：使用 `Vercel AI SDK` 实现 Server-Sent Events (SSE)，文字逐字浮现，掩盖等待延迟，增强“宇宙书写”神秘感。
- **音频懒加载**：环境音（432Hz颂钵）仅在进入第二页时加载，避免首屏阻塞。

## 2. 核心交互与视觉开发规范

### 2.1 “黑神话级”视觉实现指南

| 元素 | 技术要求 | 性能优化 |
|------|----------|----------|
| **星空背景** | GPU粒子系统 (Custom Shader) | 使用 `instancedMesh` 渲染 5000+ 星点，限制在 30fps |
| **龟壳模型** | PBR材质 (Normal+Roughness Map) | 面数控制在 2万以内，烘焙光照贴图 |
| **爻线特效** | 自定义 Shader (Glow + Edge) | 后处理 Bloom 强度调至 0.6，避免过度泛光 |
| **光影** | Environment Map (HDRI) | 使用低分辨率 HDRI (1k)，减少内存占用 |

### 2.2 交互逻辑流程图
```
入口页 (星空) → 点击"进入通道" → 转场动画(镜头推进) 
→ 龟壳页(悬浮) → 长按3秒(注入意图) → 震动反馈+裂纹发光
→ 点击6次(摇卦) → 每次生成一爻(音效+粒子)
→ 六爻完成 → 龟壳消散/卦象显现 → AI请求后台触发
→ 免费解读页(宇宙之镜) → 付费墙 → 解锁完整启示
```

**关键体验指标：**
- 摇卦总时长：≥ 45秒（不可跳过）
- 每次点击反馈延迟：< 100ms
- 音频与视觉同步误差：< 50ms

## 3. 64卦内容生产SOP

采用**“先白话重构，再身心灵翻译”**的三步法，确保内容既保留东方内核，又符合欧美语境。

### 3.1 内容重构工作流

| 步骤 | 执行人 | 产出 | 质量标准 |
|------|--------|------|----------|
| **① 原文提取** | 内容专员 | 64卦原始卦辞、象辞 | 来源权威（如《周易译注》） |
| **② 现代白话转译** | 内容专员/AI | 现代中文解读文档 | 通俗易懂，心理学视角 |
| **③ 英文身灵翻译** | AI+母语校对 | 英文内容库 | 符合New Age语境，无Chinglish |
| **④ 结构化入库** | 开发 | Supabase JSON表 | 字段完整（卦名/能量词/诊断/建议/时机） |

### 3.2 Prompt 模板示例
```text
You are a master of I Ching and a modern spiritual guide.
Input Hexagram: [Name & Text]
Tasks:
1. Explain it in Modern Chinese as a psychological diagnosis: Energy state, Core block, Action advice.
2. Translate to English New Age language. Use words: Alignment, Frequency, Manifestation, Grounding.
3. Format as JSON: {hexagram: "", english_name: "", energy_state: "", cosmic_mirror: "", natures_guidance: [], divine_timing: ""}
Tone: Mystical, empowering, poetic, actionable.
```

## 6. 风险管理与应对预案

| 风险 | 概率 | 影响 | 应对预案 |
|------|------|------|----------|
| **3D动效性能差** | 中 | 高 | 降级方案：WebGL转Canvas 2D，保留光影但简化粒子。 |
| **AI解文质量不稳** | 中 | 高 | 人工抽检+Fallback预设文本+Prompt迭代优化。 |
| **广告审核拒审** | 低 | 中 | 文案包装为“自我探索/正念”，避免“预测/算命”字眼。 |
| **冷启动流量不足** | 高 | 中 | 提前1个月铺设SEO内容，利用Reddit/Pinterest长尾引流。 |
| **支付失败率高** | 低 | 高 | 接入Stripe Radar防欺诈，提供多币种结算。 |
