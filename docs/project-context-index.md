## 读取顺序

新对话开始时，优先按以下顺序读取：

1. `docs/project-context-index.md`
2. `docs/module-summaries/home.md`
3. `docs/module-summaries/oracle-ritual.md`
4. `docs/dao-oracle-effect-notes.md`

如果本次任务只涉及某个模块，优先读取对应模块摘要，再读取相关设计规格。

## 当前核心项目定位

- 品牌名：Dao Oracle
- Slogan：Ancient Eastern wisdom for modern life decisions.
- 产品不是传统算命网站。
- 产品不是学术型易经查询网站。
- 产品是面向欧美市场的东方神秘智慧产品。
- 底层体系使用易经 64 卦。
- 前端表达重点是个人洞察、关系判断、行动建议和仪式感。
- 视觉方向要现代、克制、可信，避免廉价玄学感。

## 摘要维护规则

每次出现新的明确决定时，必须同步更新对应模块摘要。  
每日总结记录过程，模块摘要记录当前最终结论。

更新原则：

- 新决定覆盖旧决定时，要写明“已替代”的旧方向。
- 未最终确认的想法不要写成结论，要放入“待确认”。
- 页面文案、入口、视觉方向、交互动作都要写清楚。
- 新对话需要执行页面修改时，以模块摘要为准。
