# Dao Oracle 效果图讨论摘要

本文件用于后续继续讨论本项目效果图时优先读取。  
目标是把已经确认的结构、首页叙事、宇宙摇卦页气质和标注方式固定下来，避免每次重新从头梳理。

当前对应的 HTML 效果图稿：
- `docs/dao-oracle-effect-mockups.html`

## 已确认页面结构

1. 首页
2. 问题页
3. 提交后的过渡动画状态
4. 宇宙摇卦页面
5. 解卦页面
6. 付费详情页面
7. 分享页面

说明：
- 第 3 项不做成独立页面，只是问题页进入摇卦页之间的短动画。
- 效果图里可以单独画出来，但实现层面按动画状态理解。

## 首页方向

首页的目标不是讲完整历史，而是让用户一眼理解：
- 64 卦是什么
- 它为什么能给出答案
- 用户接下来要做什么

推荐表达：
- `64卦，是延续三千年的宇宙自然启示语言。`
- `The 64 hexagrams have served as a symbolic language of guidance from nature and the cosmos.`

首页建议保留一个代表性卦象图形，用来建立认知：
- 推荐先用 `坤卦 ䷁`
- 理由：更安静、更接收式，也更贴合“接收启示”的语义

首页首屏建议要素：
- 品牌名
- 一个代表性卦象
- 短标题
- 一句解释
- 开始解卦 CTA

## 首页文案原则

保留“3000 年延续”的核心叙事，但不要写成冗长历史故事。

不建议：
- “部落内拥有神秘力量的法师”
- 过强的奇幻/神秘学措辞

建议：
- 强调 ancient East / nature / cosmos / guidance
- 把卦象说成“symbolic language of guidance”
- 保持高级、克制、清晰

## 宇宙摇卦页面方向

第四页要做成：
- 空旷
- 辽阔
- 深邃
- 简洁
- 有“宇宙深处传来呼吸”的感觉

主视觉建议：
- 中央黑洞感 / 宇宙尽头感
- 呼吸波纹从中心缓慢扩散
- 最后卦象淡入

不建议：
- 把完整八卦环作为主视觉

理由：
- 对欧美市场来说，完整八卦环会显得符号密度太高
- 它更像“解释元素”，不适合当摇卦页第一视觉

折中方案：
- 可以保留非常轻的东方符号暗示
- 例如极淡的方位刻痕、微弱环线、隐约符号
- 但不能盖过深空和呼吸感

## 对欧美市场的判断

更受欢迎的方向：
- 宇宙深空感
- 黑洞/门洞感
- 呼吸脉冲
- 卦象最终浮现

较弱的方向：
- 符号堆叠
- 八卦盘直接铺满
- 过于“东方装饰化”

结论：
- 摇卦页主视觉优先用“宇宙呼吸”
- 八卦环只做轻暗示，不做主角

## 标注方式

后续做效果图时，每张图都应带旁注解释：
- 这个区域的视觉目的
- 这个元素在用户认知里的作用
- 与当前首页实现的差别

推荐每张效果图至少标注：
- 页面名称
- 关键视觉点
- 文案说明
- 与现有页面的差异

## 效果图制作标准

后续效果图应从欧美主流移动端产品设计师的角度制作，而不是只做东方玄学风格展示。

页面尺寸：
- 按主流手机页面比例设计
- 建议以 iPhone 主流宽度为基准，例如 390px 或 393px 宽
- 每张效果图都要明确安全边距、首屏重心和主要交互区域

每张页面需要准确标注：
- 页面标题
- 页面实际展示文案
- CTA 文案
- 主要视觉元素的位置
- 字体层级
- 组件间距
- 页面旁边的中文解释

文字要求：
- 每张页面都要展示准确英文文案
- 旁注使用中文，方便后续讨论、调整和前端实现
- 页面里的英文要尽量接近最终上线文本，不使用占位文案

排版要求：
- 明确主标题、副标题、说明文、按钮、辅助提示的层级
- 避免页面过满
- 保留足够呼吸感
- 重点页面，尤其是宇宙摇卦页，要简洁、空旷、有深度

前端交付辅助：
- 标注应帮助前端理解布局和交互，而不是只解释审美
- 标注内容要说明这个元素为什么存在、用户看到它应理解什么
- 后续做前端页面时，可直接依据效果图里的文字、位置、层级和标注实现

## 当前结论

当前最重要的共识是：
- 首页要先让人懂 64 卦
- 摇卦页要做成宇宙深处的呼吸感
- 八卦环不能喧宾夺主
- 效果图要带旁注，方便下次快速继续
- 效果图必须按主流手机尺寸、准确文案、清晰排版和中文旁注来制作
